# 🚀 سامانه هوشمند پرسش از اسناد (Roshan-AI)

این پروژه یک سیستم پیشرفته **RAG (Retrieval-Augmented Generation)** است که با استفاده از **Django** و **LangChain** طراحی شده است. هدف این سامانه، مدیریت اسناد متنی، جستجوی هوشمند در میان آن‌ها و تولید پاسخ‌های دقیق بر اساس دانش موجود در پایگاه داده است.

---

## ✨ قابلیت‌های کلیدی

* 🔍 **جستجوی پیشرفته فارسی**
  استفاده از `PostgreSQL Full-Text Search` به همراه کتابخانه **Hazm** برای نرمال‌سازی متن و استخراج کلمات کلیدی

* ⚖️ **رتبه‌بندی وزنی (Weighted Search)**
  اولویت‌بندی نتایج جستجو بر اساس:

  * عنوان (وزن A)
  * برچسب‌ها (وزن B)
  * محتوا (وزن C)

* 🤖 **پاسخ‌دهی هوشمند با LLM**
  استفاده از مدل **Llama-3.3-70b-versatile** از طریق زیرساخت **Groq** برای تولید پاسخ‌های سریع و کاملاً فارسی

* 🛡 **سیستم پایداری (Fallback)**
  در صورت بروز خطا در ارتباط با مدل زبانی، سیستم به‌صورت خودکار بخش‌هایی از اسناد مرتبط را به‌عنوان پاسخ نمایش می‌دهد

* 🐳 **استقرار با Docker**
  آماده‌سازی کامل پروژه با استفاده از **Docker** و **Docker Compose** برای اجرای پایدار و ساده

---

## 🛠 پشته تکنولوژی (Tech Stack)

* **Backend:** Django 6.0
* **Database:** PostgreSQL (با قابلیت Full-Text Search)
* **NLP:** کتابخانه Hazm (پردازش زبان فارسی)
* **LLM Framework:** LangChain
* **AI Model:** Llama 3.3 (از طریق Groq API)
* **Infrastructure:** Docker & Docker Compose

---

## 🏗 ساختار پروژه بر اساس فازهای اجرایی

### 📦 فاز ۱: هسته داده و پنل مدیریت

در این فاز، ساختار اصلی داده‌ها طراحی شده است:

* مدل `Document` برای ذخیره‌سازی دانش و اسناد
* مدل `QuestionResponse` برای ثبت تاریخچه پرسش و پاسخ‌ها
* بهینه‌سازی پنل مدیریت Django برای مدیریت اسناد و بررسی سوالات کاربران

---

### 🔎 فاز ۲: بازیابی اسناد مرتبط (Retrieval)

پیاده‌سازی منطق جستجو با استفاده از:

* `SearchVector`
* `SearchRank`

فرآیند:

1. نرمال‌سازی سوال کاربر با استفاده از NLP
2. جستجو در پایگاه داده
3. استخراج مرتبط‌ترین اسناد

در این مرحله از `websearch` نیز برای درک بهتر سوالات محاوره‌ای استفاده شده است.

---

### 🧠 فاز ۳: یکپارچه‌سازی با LangChain (Generation)

در این فاز:

* یک زنجیره (Chain) هوشمند طراحی شده است
* اسناد بازیابی‌شده به‌عنوان **Context** به مدل زبانی ارسال می‌شوند
* پاسخ با لحنی حرفه‌ای تولید و در سیستم ذخیره می‌شود

---

### 🚀 فاز ۴: آماده‌سازی نهایی و استقرار

* تنظیم و بهینه‌سازی فایل‌های Docker
* آماده‌سازی پروژه برای اجرا در محیط‌های مختلف بدون پیچیدگی

---

 

## 🚀 راهنمای نصب و اجرا

۱. ابتدا مخزن را کلون کنید:

Bash

```
git clone https://github.com/ALIshabani77/Roshan.git
cd Roshan

```

۲. فایل `.env` را در ریشه پروژه بسازید و کلید API خود را در آن قرار دهید:

تکه‌کد

```
GROQ_API_KEY=your_api_key_here

```

۳. پروژه را با استفاده از داکر اجرا کنید:

Bash

```
docker-compose up --build

```

۴. پس از بالا آمدن کانتینرها، از طریق آدرس `[http://127.0.0.1:8000/ask/](http://127.0.0.1:8000/ask/)` می‌توانید سوالات خود را بپرسید.

---

## 💡 نکات فنی و خلاقیت‌ها

* **جستجوی منعطف:** استفاده از `search_type='websearch'` برای درک بهتر سوالات محاوره‌ای کاربران.
* **سیستم Fallback:** در صورت قطع ارتباط با API هوش مصنوعی، سیستم به طور خودکار بخشی از مرتبط‌ترین سند یافت شده را به عنوان پاسخ مستقیم نمایش می‌دهد تا تجربه کاربری مختل نشود.
* **مدیریت حافظه در Prompt:** محدود کردن حجم Context ارسالی به ۳۰۰۰ کاراکتر برای بهینه‌سازی سرعت و هزینه.
